**Project Overview**

This repository contains the complete documentation, supporting resources, and final report for **Ethical Hacking – Task 01**, completed as part of the **White Band Associates Cyber Security Internship Programme**.

The objective of this project was to perform a **Passive Reconnaissance (Open Source Intelligence – OSINT)** assessment against the official website of **Kristu Jayanti University**. The assessment focused on collecting and analysing publicly available information to understand the target's internet-facing infrastructure, technology stack, and overall security posture.

Unlike active penetration testing, this assessment was conducted entirely through passive information gathering. No vulnerability scanning, exploitation, authentication attempts, brute-force attacks, or unauthorized access were performed during any stage of the project. All observations documented in this repository were obtained using publicly accessible resources while adhering to ethical hacking principles.

---

**Project Objectives**

The primary objectives of this assessment were to:

* Understand the reconnaissance phase of the ethical hacking lifecycle.
* Perform passive information gathering using Open Source Intelligence (OSINT) techniques.
* Analyse publicly available domain registration information.
* Enumerate DNS records to understand the target's infrastructure.
* Identify hosting services and website technologies.
* Evaluate the implementation of HTTP security headers.
* Examine publicly accessible configuration files.
* Document technical findings in a structured and professional security report.

**Tools Used**

| Wappalyzer      | Website technology fingerprinting          |

These tools were selected because they provide valuable intelligence while ensuring the assessment remains completely passive and non-intrusive.

---

**Assessment Workflow**

The project followed a structured reconnaissance methodology commonly used during the initial phase of professional security assessments.

1. Target Selection
2. WHOIS Analysis
3. DNS Enumeration
4. Website Technology Identification
5. HTTP Security Header Assessment
6. robots.txt Analysis
7. sitemap.xml Analysis
8. Security Observation
9. Report Preparation

Each stage contributed to building a comprehensive understanding of the target's publicly exposed infrastructure and external security posture.

---

**Summary of the Assessment**

The reconnaissance exercise successfully collected information regarding the target's domain registration, DNS configuration, hosting infrastructure, web technologies, browser security configuration, and publicly accessible resources.

The assessment confirmed that valuable intelligence can be obtained using only passive reconnaissance techniques, highlighting the importance of information gathering before conducting any active security testing.

The accompanying report provides detailed analysis supported by screenshots, observations, and technical explanations for every stage of the assessment.

---

**Key Findings**

The assessment identified several noteworthy observations, including:

* Publicly available domain registration information.
* DNS infrastructure and cloud-based hosting configuration.
* Website technology stack and supporting frameworks.
* Public DNS records including A, MX, NS, and TXT records.
* Browser security header implementation.
* Publicly accessible configuration files such as **robots.txt** and **sitemap.xml**.
* Areas where software maintenance and security hardening may improve the overall security posture.

These findings demonstrate how publicly available information can contribute to understanding an organization's digital footprint without directly interacting with its systems.
